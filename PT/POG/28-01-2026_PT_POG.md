## Einstieg in die OOP mit Herrn POG

### Was ist OOP?
- **Objektorientierte Programmierung**
- Ein **Programmierparadigma**
- Zentrale Konzepte:
  - Klassen
  - Objekte
  - Vererbung
  - Kapselung
  - Polymorphismus
  - Abstraktion

### Was ist ein Objekt?
- **Instanz einer Klasse**
- Besteht aus:
  - **Eigenschaften (Attribute)**
  - **Verhalten (Methoden)**
- Sammlung von Daten (Attributen) und zugehörigen Methoden

#### Unterschied zwischen Methode und Funktion
- **Methode**
  - An ein Objekt bzw. eine Klasse gebunden
- **Funktion**
  - Eigenständig, nicht an ein Objekt gebunden

#### Was ist eine Funktion?
- Wiederverwendbarer Codeblock
- Nimmt Eingaben entgegen (**Parameter**)
- Liefert ein Ergebnis zurück (**Return-Wert**)

### Wichtige Prinzipien der OOP
- **Kapselung**
  - Daten und Methoden werden in einem Objekt zusammengefasst
- **Modularität**
  - Code wird in kleine, wiederverwendbare Einheiten aufgeteilt
- **Vererbung**
  - Klassen können Eigenschaften und Methoden von anderen Klassen erben
- **Polymorphismus**
  - Gleiche Methode kann in verschiedenen Klassen unterschiedlich implementiert werden
- **Abstraktion**
  - Reduktion von Komplexität durch Fokus auf wesentliche Eigenschaften

**Grundidee:**  
Eine Klasse wird einmal programmiert und dient anschließend als Vorlage für viele Objekte  
(Beispiel: *Bäcker* = Klasse, *Brötchen* = Objekt)

### Vorteile der OOP
- Strukturierter Code
- Hohe Wiederverwendbarkeit
- Gute Wartbarkeit
- Nähe zur Realwelt-Modellierung

---

## Klassen
- **Bauplan für Objekte**
- Definieren:
  - Eigenschaften (Attribute)
  - Verhalten (Methoden)
- **Instanziierung**
  - Erzeugen eines Objekts aus einer Klasse

### Aufbau einer Klasse
- Klassenname
- Attribute (Instanzvariablen)
- Methoden (Funktionen)
- Konstruktor
  - Initialisierungsmethode
  - In Python: `__init__`
  - Wird beim Erstellen eines Objekts automatisch aufgerufen
  - Initialisiert die Attribute des Objekts

## Objekte
- **Instanzen von Klassen**
- Repräsentieren konkrete Ausprägungen des Klassen-Bauplans
