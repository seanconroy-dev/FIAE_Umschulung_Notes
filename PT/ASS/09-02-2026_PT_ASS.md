## Konstruktoren

### Arten von Konstruktoren

- **Default-Konstruktor**
  - Konstruktor ohne Parameter
  - Wird automatisch vom Compiler erzeugt, **nur wenn kein anderer Konstruktor definiert ist**
  - Kann auch explizit definiert werden

- **Konstruktor mit Übergabeparametern**
  - Initialisiert Objektzustand beim Erzeugen des Objekts
  - Ermöglicht kontrollierte Initialwerte

---

## Überladen von Konstruktoren (Constructor Overloading)

- Mehrere Konstruktoren in derselben Klasse
- Unterscheidung erfolgt **ausschließlich über die Parameterliste**

Mögliche Unterschiede:
- Unterschiedliche Anzahl von Parametern
- Gleiche Anzahl von Parametern, aber unterschiedliche Datentypen

Nicht relevant für Overloading:
- Rückgabetyp (Konstruktoren haben keinen)
- Parameternamen allein

---

## Allgemeine Regeln für Constructor Overloading

1. Jeder Konstruktor muss eine eindeutige Parameterliste besitzen.
2. Der Konstruktorname ist immer identisch mit dem Klassennamen.
3. Konstruktoren haben **keinen Rückgabetyp**, auch nicht `void`.
4. Konstruktorverkettung (Constructor Chaining):
   - Ein Konstruktor kann einen anderen Konstruktor derselben Klasse aufrufen.
   - Dafür wird das Schlüsselwort `this()` verwendet.
   - `this()` muss **immer die erste Anweisung** im Konstruktor sein.

---

## Beispiel: Constructor Chaining mit Gettern und Settern

public class Person {
    private String name;
    private int age;

    // Default-Konstruktor
    public Person() {
        this("Unknown", 0); // ruft den parameterisierten Konstruktor auf
    }

    // Konstruktor mit Parametern
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // Getter und Setter
    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        this.age = age;
    }
}

Wichtige Beobachtungen:
- Objektinitialisierung erfolgt zentral im parameterisierten Konstruktor.
- Reduziert doppelten Code.
- Default-Konstruktor delegiert Verantwortung weiter.
