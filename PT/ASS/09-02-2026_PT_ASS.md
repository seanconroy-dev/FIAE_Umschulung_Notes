## JAVA OOP

### Getter und Setter in Java
**Zweck:**
- Ermöglichen kontrollierten Zugriff auf private Attribute
- Umsetzung des Prinzips der Kapselung (Encapsulation)
- Validierung von Eingaben vor dem Setzen von Werten
- Steigern Wartbarkeit und Lesbarkeit des Codes

---

## Konstruktoren

### Arten von Konstruktoren

| Konstruktor-Typ                | Beschreibung                                                                                     |
|--------------------------------|-------------------------------------------------------------------------------------------------|
| **Default-Konstruktor**        | Parameterlos; wird automatisch generiert, **nur wenn kein anderer Konstruktor definiert ist**   |
| **Parameterisierter Konstruktor** | Initialisiert Objektattribute direkt bei der Objekterzeugung; ermöglicht kontrollierte Startwerte |

---

## Überladen von Konstruktoren (Constructor Overloading)

- **Definition:** Mehrere Konstruktoren in einer Klasse, die sich durch ihre Parameterliste unterscheiden
- **Merkmale:**
  - Unterschiedliche Anzahl oder Datentypen der Parameter
  - Rückgabetyp und Parameternamen allein sind **nicht** relevant
- **Regeln:**
  1. Jeder Konstruktor muss eine **eindeutige Parameterliste** besitzen
  2. Der Name entspricht immer dem Klassennamen
  3. Konstruktoren haben **keinen Rückgabetyp** (auch nicht `void`)

---

## Konstruktorverkettung (Constructor Chaining)

- Ein Konstruktor ruft einen anderen Konstruktor derselben Klasse auf
- Verwendung von `this()` als **erste Anweisung** im Konstruktor
- **Vorteile:**
  - Zentrale Initialisierungslogik
  - Vermeidung von Code-Duplikation

---

## Beispiel: Constructor Chaining mit Gettern und Settern

```java
public class Person {
    private String name;
    private int age;

    // Default-Konstruktor
    public Person() {
        this("Unknown", 0); // Delegation an parameterisierten Konstruktor
    }

    // Parameterisierter Konstruktor
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // Getter und Setter
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
    public int getAge() { return age; }
    public void setAge(int age) { this.age = age; }
}
```

**Wichtige Beobachtungen:**
- Initialisierung erfolgt zentral im parameterisierten Konstruktor
- Default-Konstruktor delegiert Verantwortung
- Reduziert redundanten Code
