## Objektorientierte Programmierung (OOP)

### Grundidee
- Objektorientierte Programmierung ist ein Programmierparadigma
- Programme bestehen aus Objekten, die miteinander interagieren

### Objekte
- Ein Objekt repräsentiert etwas aus der realen oder abstrakten Welt
- Grundsatz: Alles, was existiert, kann als Objekt modelliert werden
- Ein Objekt benötigt immer eine Klasse
- Objekte werden z. B. mit `new` erzeugt

### Klassen
- Eine Klasse ist der Bauplan für ein Objekt
- Sie definiert:
  - Attribute (Eigenschaften)
  - Datentypen der Attribute
- Jede Klasse stellt einen eigenen Datentyp dar

### Attribute
- Attribute sind Variablen innerhalb einer Klasse
- Sie beschreiben die Eigenschaften eines Objekts
- Es gibt:
  - vorhandene (primitive) Datentypen
  - nicht vorhandene / selbst definierte Datentypen (z. B. andere Klassen)

### Typen von Klassen
- Entity-Klassen  
  - Repräsentieren Datenobjekte (z. B. Datenbankeinträge)
- Model-Klassen  
  - Dienen der fachlichen Modellierung von Daten und Logik

## Hibernate (ORM – Object Relational Mapping)

### Zweck
- Hibernate ist ein ORM-Tool
- Es verbindet objektorientierte Klassen mit relationalen Datenbanken

### Zentrale Konzepte
- Mapping  
  - Zuordnung von Klassen zu Datenbanktabellen
- Mapping-Dateien  
  - Beschreiben die Abbildung von Klassen und Attributen
- XML-Dateien  
  - Werden für Konfiguration und Mapping verwendet
- `hibernate.cfg.xml`  
  - Zentrale Konfigurationsdatei von Hibernate
