4000 → 7FFF
8000 → BFFF
C000 → FFFF

------------------------------------------------------------
Split into 8 (/67)
Step size = 2000

0000 → 1FFF
2000 → 3FFF
4000 → 5FFF
6000 → 7FFF
8000 → 9FFF
A000 → BFFF
C000 → DFFF
E000 → FFFF

------------------------------------------------------------
Split into 16 (/68)
Step size = 1000

0000 → 0FFF
1000 → 1FFF
2000 → 2FFF
…
F000 → FFFF
This compact view helps most learners because it reveals the pattern instantly.

How to remember this for a test
Here’s the key trick:

Look only at the first hex digit of the 5th block.
Ignore everything else.

Then apply this:

/65 = cut hex 0–F into two halves: 0–7 and 8–F

/66 = 4 chunks: 0–3, 4–7, 8–B, C–F

/67 = 8 chunks: step size 2

/68 = 16 chunks: step size 1

Everything in the right column becomes all Fs.
Everything else stays the same.

Once you see it that way, it becomes trivial instead of huge.

Lernhilfe:
Der gesamte Trick beim IPv6-Subnetting steckt in der 5. Gruppe.  
Sobald du ein /64 hast, liegt der komplette Hostbereich in den letzten vier Blöcken.  
Wenn du also Subnetze erzeugst, arbeitest du NUR in diesem einen Block — Hex macht dabei alles sauber sichtbar.

Merkregel:
- /65 = Halbierung → 0–7 und 8–F  
- /66 = Viertelung → 0–3, 4–7, 8–B, C–F  
- /67 = Achteln → Schritte 2 (0,2,4,6,8,A,C,E)  
- /68 = Sechzehnteln → Schritte 1 (0–F)

Wenn du diese Muster erkennst, ist jede IPv6-Subnetzrechnung in 10 Sekunden möglich.

## Zusatz: Begriffe für Prüfung & Verständnis

### IPv4 Class B

Historisches Konzept aus der „classful“ IPv4-Welt (wird in Prüfungen noch gerne abgefragt):

- **Class B Adressen**:
  - Bitmuster am Anfang: `10xxxxxx`
  - Adressbereich: **128.0.0.0 – 191.255.255.255**
  - Standard-Netzmaske: **255.255.0.0** = **/16**
- Typische Prüfungsaussagen:
  - „Class B – mittleres Netz, viele Hosts“
  - Ca. **16.384 Netze** × **65.534 Hosts** (theoretisch, ohne Reservierungen)
- Praxis heute:
  - Wir benutzen **CIDR** (Classless Inter-Domain Routing), also z.B. /20, /27 usw.
  - Die Begriffe „Class A/B/C“ sind **noch Prüfungsstoff**, aber technisch veraltet.

**Merksatz:**  
_Class B = 128–191, Maske /16, früher viele Hosts pro Netz – heute nur noch im Geschichtsbuch und in Prüfungen wichtig._

---

### Link Local & APIPA – „eine Lösung, die eigentlich keine ist“

#### APIPA (IPv4)

**APIPA = Automatic Private IP Addressing**  
Das ist das Verhalten vieler Betriebssysteme (besonders Windows), wenn **kein DHCP-Server** antwortet:

- Adressbereich: **169.254.0.0/16**
- Wird automatisch gesetzt, wenn:
  - IP auf „automatisch beziehen“ steht (DHCP)
  - aber **kein** DHCP-Server erreichbar ist
- Eigenschaften:
  - Gültig **nur im lokalen Layer-2-Netz** (Broadcast-Domain)
  - **Nicht routbar** über Router hinweg
  - Kommunikation nur mit Geräten, die auch eine 169.254.x.x-Adresse haben

Deshalb:  
> „eine Lösung, die eigentlich keine ist“  
Du hast zwar eine IP, aber **kein echtes Netz** – typischerweise kein Internet, kein Routing, nur Notbetrieb.

**Merksatz:**  
_APIPA = 169.254.x.x, bedeutet: DHCP kaputt, nur Notbetrieb im eigenen Netzsegment._

---

#### Link-Local (IPv6)

IPv6 kennt ebenfalls Link-Local-Adressen – aber **die sind Pflicht und sinnvoll**, nicht nur Notlösung:

- Präfix: **fe80::/10**  
  (typisch dargestellt als `fe80::/64` im lokalen Segment)
- Jede IPv6-Schnittstelle hat **immer** eine Link-Local-Adresse
- Gültig nur im **lokalen Link** (nicht geroutet)
- Wichtige Rollen:
  - **Neighbour Discovery** (Nachbarn finden, MAC-Adressen lernen)
  - **Router Discovery** (Router Solicitation / Router Advertisement)
  - Wird als **Next-Hop** für Gateways verwendet (Default Gateway ist oft Link-Local)

**Merksatz:**  
_Link-Local (IPv6) = fe80::/10, immer da, nicht geroutet, aber absolut notwendig für alles andere._

---

### SLAAC – Stateless Address Autoconfiguration (IPv6)

**SLAAC** erlaubt einem IPv6-Host, sich **selbst** eine gültige Adresse zu geben – ohne klassischen DHCP-Server, aber mit Hilfe von Router-Infos.

#### Idee

- „Stateless“ = Der Router speichert **keinen Zustand** zu einzelnen Clients.
- Der Host baut sich seine Adresse aus:
  - dem Präfix vom Router (**z.B. 2001:db8:1234:1::/64**)
  - und einem Interface-Identifier (**z.B. zufällig oder aus der MAC-Adresse**)

#### Ablauf (vereinfacht)

1. Host generiert eine **Link-Local-Adresse** (`fe80::...`).
2. Host sendet eine **Router Solicitation (RS)** an das Netz.
3. Router antwortet mit einem **Router Advertisement (RA)**:
   - enthält u.a.:
     - Präfix (z.B. `2001:db8:1234:1::/64`)
     - Flags (z.B. „SLAAC erlaubt“, „DHCPv6 nutzen ja/nein“)
4. Host:
   - kombiniert Präfix + Interface-ID → baut **Global Unicast Adresse**
   - trägt Default-Gateway (Link-Local-Adresse des Routers) ein
   - optional: bekommt DNS per RA oder zusätzlich per **DHCPv6**

#### SLAAC vs. DHCPv6 (prüfungsrelevant)

- **SLAAC (stateless)**:
  - Router kennt keine Liste aller Clients
  - Einfach, wenig Overhead
  - Gut für „Plug & Play“ Netze
- **DHCPv6 (stateful)**:
  - Server verwaltet, wer welche Adresse bekommt
  - Besser für zentrale Verwaltung / Logging / Policies

**Merksätze:**

- _SLAAC = der Host baut sich seine Adresse selbst, mit Infos vom Router._  
- _„Stateless“ = Router merkt sich nicht, wer welche Adresse genommen hat._  
- _IPv6 ohne SLAAC / Link-Local / RA ist wie ein Auto ohne Zündung – hübsch, aber fährt nicht._
