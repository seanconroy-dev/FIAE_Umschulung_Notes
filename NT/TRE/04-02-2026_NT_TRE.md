# Schicht 4: Transport (OSI-Modell)
===============================

## OSI-Schicht 4: Transport
-----------------------
- **Aufgabe:** Verantwortlich für die **Ende-zu-Ende-Kommunikation** zwischen Hosts
- **Funktionen:**
  - Gewährleistet die **Zuverlässigkeit der Datenübertragung**
  - Segmentierung und Wiederzusammenführung von Daten
  - Fehlererkennung und -behebung
  - Flusskontrolle und Staukontrolle

---

## Ports
-------
- **Definition:** Ports sind **logische Endpunkte** in der Transportschicht (Schicht 4), die Anwendungen und Dienste auf einem Host identifizieren.
- **Zweck:** Ermöglichen die Kommunikation zwischen Prozessen auf verschiedenen Hosts und den Anwendungsschichten (5-7).
- **Port-Kategorien:**
  | Kategorie               | Port-Bereich   | Verwendung                                                                 |
  |-------------------------|---------------|----------------------------------------------------------------------------|
  | **Well-known Ports**    | 0–1023        | Standardisierte Dienste (z. B. HTTP, FTP)                                 |
  | **Registered Ports**    | 1024–49151    | Registrierte Anwendungen (z. B. MySQL, PostgreSQL)                         |
  | **Dynamic/Private Ports**| 49152–65535   | Dynamisch zugewiesene Ports für temporäre Verbindungen (z. B. Client-Ports) |

- **Verwendung:**
  - Ports werden in **TCP/UDP-Headern** angegeben.
  - Ports können **statisch** (fest zugewiesen) oder **dynamisch** (bei Bedarf) vergeben werden.

- **Beispiele für Portnummern:**
  | Dienst/Protokoll       | Port(s)                     |
  |------------------------|----------------------------|
  | FTP                    | 20 (Daten), 21 (Steuerung) |
  | SSH                    | 22                         |
  | Telnet                 | 23                         |
  | SMTP                   | 25                         |
  | DNS                    | 53                         |
  | DHCP                   | 67 (Server), 68 (Client)   |
  | HTTP                   | 80                         |
  | POP3                   | 110                        |
  | IMAP                   | 143                        |
  | HTTPS                  | 443                        |
  | RDP                    | 3389                       |
  | MySQL                  | 3306                       |
  | PostgreSQL             | 5432                       |
  | MongoDB                | 27017                      |
  | Redis                  | 6379                       |
  | Memcached              | 11211                      |
  | RabbitMQ               | 5672                       |
  | NTP                    | 123                        |
  | Drucker (Standard)     | 9100 (TCP)                 |

---

## TCP (Transmission Control Protocol)
-----------------------
- **Eigenschaften:**
  - **Verbindungsorientiert:** Baut eine Verbindung vor der Datenübertragung auf
  - **Zuverlässig:** Garantiert die korrekte Zustellung der Daten
  - **Mechanismen:**
    - **Sequenznummern** und **Bestätigungen (ACKs)** für Fehlererkennung und -korrektur
    - **Flusskontrolle** und **Staukontrolle** zur Optimierung der Datenübertragung
  - **Anwendungsbeispiele:** HTTP, FTP, SMTP, HTTPS

---

## UDP (User Datagram Protocol)
-----------------------
- **Eigenschaften:**
  - **Verbindungslos:** Kein Verbindungsaufbau erforderlich
  - **Unzuverlässig:** Keine Garantie für die Zustellung der Daten
  - **Vorteile:**
    - **Geringer Overhead:** Schnelle Datenübertragung
    - **Keine Fehlerkorrektur oder Flusskontrolle**
  - **Anwendungsbeispiele:** DNS, DHCP, VoIP, Video-Streaming, Online-Gaming

TCP (Transmission Control Protocol)
-----------------------
- Verbindungsorientiertes Protokoll
- Stellt eine zuverlässige Datenübertragung sicher
- Verwendet Sequenznummern und Bestätigungen (ACKs) zur Fehlererkennung und -korrektur
- Implementiert Flusskontrolle und Staukontrolle
- Beispielanwendungen: HTTP, FTP, SMTP, HTTPS

UDP (User Datagram Protocol)
-----------------------
- Verbindungsloses Protokoll
- Bietet keine Garantie für die Zustellung von Daten
- Geringer Overhead, schneller als TCP
- Keine Fehlerkorrektur oder Flusskontrolle
- Verwendet keine Verbindungsaufbau- oder Abbauprozesse
- Geeignet für Anwendungen, die Echtzeitübertragung erfordern
- Beispielanwendungen: DNS, DHCP, VoIP, Video-Streaming, Online-Gaming
