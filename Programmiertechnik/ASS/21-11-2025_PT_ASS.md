
## Freitag, 21-11-2025_PT_ASS  
### Methoden, Sichtbarkeit und Zugriff (Grundlagen)

**Thema:** Methoden, Zugriffsmodifizierer, Packages  
**Kontext:** Objektorientierte Programmierung (z. B. Java)

---

## 1. Was ist eine Methode?

Eine **Methode** ist ein benannter Programmblock, der:
- eine bestimmte Aufgabe ausführt
- optional Werte entgegennimmt (Parameter)
- optional einen Wert zurückgibt (Returnwert)

Methoden dienen dazu:
- Code zu strukturieren
- Wiederholungen zu vermeiden
- Logik klar zu kapseln

---

## 2. Methodendefinition

Bei der **Definition einer Methode** wird festgelegt:

- **Zugriffsmodifizierer** (Sichtbarkeit)
- **Rückgabetyp**
- **Methodenname**
- **Parameterliste**
- **Methodenrumpf**

Beispiel (Java):
```java
public int add(int a, int b) {
    return a + b;
}
```

---

## 3. Methodenaufruf

Eine Methode wird über ihren **Namen** aufgerufen.

Beispiel:
```java
int ergebnis = add(3, 5);
```

Voraussetzungen:
- Die Methode ist **sichtbar**
- Die Parameter passen zu Typ und Anzahl
- Der Aufruf erfolgt im gültigen Kontext

---

## 4. Zugriffsmodifizierer (Sichtbarkeit)

Zugriffsmodifizierer bestimmen, **von wo aus** auf eine Methode zugegriffen werden darf.

### 4.1 public
- Zugriff **überall**
- Auch aus anderen Klassen und Packages

```java
public void print() { }
```

---

### 4.2 private
- Zugriff **nur innerhalb derselben Klasse**
- Nicht von außen aufrufbar

```java
private void calc() { }
```

---

### 4.3 package-private (kein Schlüsselwort)
- Zugriff **nur innerhalb desselben Packages**
- Standard, wenn kein Modifizierer angegeben ist

```java
void helper() { }
```

---

### 4.4 protected (der Vollständigkeit halber)
- Zugriff im selben Package
- Zugriff in Unterklassen (auch packageübergreifend)

```java
protected void log() { }
```

---

## 5. Überblick: Sichtbarkeit

| Modifizierer | Gleiche Klasse | Gleiches Package | Unterklasse | Anderes Package |
|-------------|---------------|------------------|-------------|-----------------|
| public      | ✓             | ✓                | ✓           | ✓               |
| protected   | ✓             | ✓                | ✓           | ✗               |
| package     | ✓             | ✓                | ✗           | ✗               |
| private     | ✓             | ✗                | ✗           | ✗               |

---

## 6. Packages – Ordnung im Projekt

Ein **Package** ist ein logischer Namensraum für Klassen  
(entspricht oft einem Ordner im Projekt).

Beispiel:
```java
package de.bfw.example;
```

Packages:
- strukturieren große Projekte
- vermeiden Namenskonflikte
- steuern Sichtbarkeit

---

## 7. Zugriff über Package-Grenzen

### Wichtige Regel
> **Methoden aus einem anderen Package sind nur mit `public` (oder `protected` bei Vererbung) zugreifbar.**

Beispiel:
```java
public void start() { }
```

Eine Methode mit:
- `package-private` → **nicht** aus anderem Package nutzbar
- `private` → **niemals** von außen nutzbar

---

## 8. Typische Fehler

- Methode ohne `public`, aber Zugriff aus anderem Package erwartet
- Verwechslung von `private` und `package-private`
- Annahme, dass Ordnerstruktur allein Zugriff regelt
- Methodenaufruf ohne passende Parameter

---

## 9. Prüfungsrelevanz (FIAE)

Typische Fragen:
- „Was legt man bei einer Methode fest?“
- „Welche Zugriffsmodifizierer gibt es?“
- „Warum ist eine Methode aus einem anderen Package nicht erreichbar?“
- „Wann verwendet man `private`?“

---

## Kernaussagen

- Methoden kapseln Verhalten
- Zugriffsmodifizierer steuern Sichtbarkeit
- Packages definieren Zugriffsräume
- Über Package-Grenzen hinweg braucht man **public**
